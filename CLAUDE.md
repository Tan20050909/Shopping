# CLAUDE.md — AI 开发约束规范

本项目是电商平台 Shopping，当前目标：**修复现有功能、统一三端逻辑、保证项目能稳定演示**。

总原则：**稳定、少改、清晰、能跑、能演示**。不要自由发挥，不要重做架构，不要新增无关功能。

---

## 1. 正式入口

- 用户端：首页、商品浏览、购物流程：`http://localhost:5173`
- 商家端登录 / 统一登录页：`http://localhost:3000/login`
- 商家后台：`http://localhost:3000/dashboard`
- 管理后台：按当前 admin 配置访问，通常是 `http://localhost:3002`

公共首页 `homepage-common` / `3001` 不再作为正式入口。禁止重新创建公共首页，禁止新增首页端口。

---

## 2. 最高禁止事项

未经明确允许，禁止：

- 新建首页、公共首页、登录页、前端项目或端口
- 重构三端架构
- 修改数据库结构
- 删除核心功能
- 大规模格式化文件
- 顺手重构无关代码
- 修改无关页面、接口、配置

发现无关问题，只能汇报，不能擅自修改。

---

## 3. 三端职责边界

用户端只负责：未登录首页、商品浏览、商品详情、店铺浏览、购物车、下单支付、订单/物流、售后、评价、优惠券、用户与商家聊天、直播浏览。禁止加入商家管理、后台管理功能。

商家端只负责：商家登录/注册、店铺资料、商品管理、库存/价格/SKU、订单处理、发货、售后处理、商家客服、优惠券/活动、直播、商家数据中心。商家端路由禁止引用用户端页面。

管理后台只负责：用户管理、商户管理、商品审核、订单监管、售后监管、纠纷仲裁、评论管理、分类/Banner/活动/优惠券管理、权限角色、通知中心、数据报表。后台禁止承载用户购物或商家经营页面。

---

## 4. 数据库约束

未经明确允许，禁止：改表结构、新增表、删除表、新增字段、删除字段、修改字段类型、清空数据、批量删除数据、覆盖正式数据。

允许：查询数据库、阅读表结构、添加少量测试数据。

如确实需要改数据库，必须先说明原因、影响范围、具体 SQL，等待确认。

---

## 5. 订单状态统一规则

订单状态必须按数据库定义：

```text
order_status:
0 待支付
1 待发货
2 待收货
3 已完成
4 已取消
5 售后中
```

支付状态单独使用：

```text
pay_status:
0 待支付
1 支付成功
2 支付失败
3 已关闭
4 退款中
5 已退款
```

禁止：

- 把 `order_status = 2` 显示成“已发货”
- 把 `order_status = 5` 显示成“已退款”
- 混用 `order_status` 和 `pay_status`
- 三端显示不同状态文案

修改订单功能时，必须同时检查用户端、商家端、管理后台。

---

## 6. 登录约束

当前暂时保留 `3000/login = 统一登录入口`。

身份必须隔离：

```text
用户登录：shopping_user_token + shopping_auth_role=user
商家登录：merchantUser + shopping_auth_role=merchant
管理员登录：只允许跳回管理后台，通过 adminToken 传递
```

禁止：用户进入商家后台、商家进入用户业务页、管理员 token 写入商家状态、用 URL 参数随意切换身份、新建登录中心或新端口。

---

## 7. 路由约束

禁止：商家端引用用户端页面、用户端引用商家端页面、后台引用用户端或商家端页面、新增 `/home2` / `/new-home` / `/portal` / `/landing`、恢复 `3001` 公共首页。

允许：修复错误跳转、修复 404、修复商家端错误跳转，例如 `/products/:id` 改为 `/goods/:id`。

---

## 8. 商家 Dashboard 约束

商家 Dashboard 只做概览和快捷入口。

允许：今日销售额、今日订单数、待发货数量、待售后数量、待审核商品、热销榜、快捷入口。

禁止：完整订单列表、发货操作、改价操作、邮费设置、复杂搜索筛选。

订单操作统一放在：`Shopping/merchant/frontend/src/views/Order.vue`。

---

## 9. 修改范围原则

每次只解决一个小问题。必须先 `git status`，再阅读相关代码，说明计划修改哪些文件，等确认后再改。修改后说明改了什么，并给出测试方式。

禁止一次性处理多个大问题。

---

## 10. Git 禁止项

禁止擅自执行：

```bash
git reset --hard
git clean -fd
git push --force
git rebase
git checkout .
```

修改后只提供建议提交信息，不要擅自 push。

---

每次完成后按这个格式汇报：

```text

1. 修改文件：
- xxx：做了什么

2. 删除文件/目录：
- 无 / xxx

3. 是否修改数据库：
- 否 / 是

4. 是否修改后端接口：
- 否 / 是

5. 是否新增页面/端口：
- 否

6. 测试方式：
- 启动命令：
- 测试页面：
- 测试步骤：
- 预期结果：
- 实际结果：

7. 风险说明：
- 是否影响已有功能：
- 需要重点检查的地方：

8. 建议提交信息：
- git commit -m "xxx"
```

AI 必须服务于现有项目，不得自由扩展项目。
