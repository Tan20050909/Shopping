# CLAUDE.md — AI 开发约束规范

本项目是一个全栈电商平台（Shopping），包含用户端、商家端、管理后台三个子系统。AI 协助开发时必须严格遵守以下规则。

---

## 项目结构速览

| 子系统 | 后端 | 前端 | 端口 |
|---|---|---|---|
| 用户端 | `Shopping/user/` (Spring Boot 3.2, Java 21) | `Shopping/user/frontend/` (Vue 3 + Vite) | 后端 8082, 前端 5173 |
| 商家端 | `Shopping/mechant/backend/` (Spring Boot 2.7, Java 17) | `Shopping/mechant/frontend/` (Vue 3 + Vite) | 后端 8081, 前端 3000 |
| 管理后台 | `admin/` (Spring Boot 3.3, Java 21) | `admin/` (Vue 3 + Vite) | 后端 8080 |
| 公共首页 | — | `Shopping/homepage-common/` (Vue 3 + Vite) | 前端 3001 |

数据库：MySQL，各子系统使用独立库（端口 3307/3308）。

---

## 1. 数据库约束（最高优先级）

**禁止操作（未经明确允许）：**
- 修改表结构（字段名、类型、长度、主键、外键、索引、唯一约束）
- 新增 / 删除表或字段
- 修改已有正式数据
- 编写会清空、覆盖、批量删除数据的 SQL

**允许操作：**
- 添加少量测试数据
- 编写查询语句用于排查问题
- 阅读现有数据库结构

**如需改表，必须：** 先说明原因、影响范围、具体 SQL，等待确认后再执行。

---

## 2. 代码修改范围约束（最小修改原则）

- 只修改与当前任务**直接相关**的代码
- 禁止顺手重构无关代码
- 禁止因为"看起来更好"而调整项目结构
- 禁止随意改动公共组件、工具类、全局样式、路由配置、权限逻辑
- 禁止删除已有功能
- 禁止改变已有接口的返回格式（除非明确允许）
- 禁止修改与当前功能无关的页面、接口、服务或配置文件

**发现无关代码有问题：** 只能记录并告知，不能直接修改。

---

## 3. 文件操作约束

**禁止操作（未经明确允许）：**
- 删除、重命名、移动文件
- 大规模格式化文件
- 修改关键配置文件：`application.yml`、`.env`、`.gitignore`、`package-lock.json`、`pom.xml`、数据库配置、部署配置
- 修改生成目录：`node_modules`、`dist`、`target`、`.idea`、`.vscode`

**如需修改配置文件：** 必须先说明原因和影响，等待确认。

---

## 4. 功能开发流程

每次开发功能，必须按以下步骤执行：

1. **先阅读相关代码**，不要立即修改
2. **说明影响范围**：本次修改涉及哪些文件
3. **给出开发方案**：简短描述实现思路
4. **等待确认**后再动手
5. **分阶段完成**：每次只完成一个小阶段
6. **修改完成后汇报**：改了哪些文件、每个文件改了什么、如何测试

**禁止一次性完成多个大功能。**

---

## 5. 测试要求

修改完成后，必须尽可能进行测试，并说明：

- **启动命令**：前端 `npm run dev`，后端 `mvn spring-boot:run`
- **测试页面**：具体访问哪个页面
- **测试步骤**：点击哪些按钮、填写什么内容
- **预期结果**：应该看到什么
- **回归影响**：是否可能影响已有功能

**如果无法运行测试，必须明确说明原因，不能假装已测试通过。**

---

## 6. Git 约束

**修改前：**
- 必须先运行 `git status` 检查状态
- 如有未提交修改，必须先告知，不能直接继续

**修改后：**
- 提供建议的提交信息，例如：`git commit -m "feat: add order search filter"`

**禁止擅自执行（除非明确要求）：**
- `git reset --hard`
- `git clean -fd`
- `git push --force`
- `git rebase`

---

## 8. 总原则

**第一原则：稳定优先。**

AI 的任务不是自由发挥，而是在现有项目基础上**小心添加功能**。所有修改必须做到：

- **可解释** — 能说清为什么这样改
- **可测试** — 能验证改完后是否正常
- **可回退** — 改坏了能轻松恢复
- **范围小** — 只动该动的
- **不破坏现有功能** — 改完后原有功能必须正常工作
