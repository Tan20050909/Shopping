# Allmart 前端风格规范

本文档定义 Allmart 用户端的视觉设计规范。当前项目定位是“品牌官网感 + 电商完整功能”，不是后台系统、低价促销商城，也不是外部品牌的仿站。

## 1. 设计定位

- 白底、浅灰分区、品牌红点题，整体干净年轻。
- 首屏强调大图 Banner、品牌感和精选内容，功能页强调清晰和效率。
- 商品、订单、购物车、优惠券、个人中心等业务页面保持信息可扫读，不做拥挤促销。
- 所有新增页面优先复用现有全局类和组件节奏，不另起一套视觉语言。

## 2. 核心文件

前端改动前优先查看：

- `frontend/src/styles.css`
- `frontend/src/App.vue`
- `frontend/src/views/Home.vue`
- `frontend/src/components/ProductCard.vue`
- 当前任务涉及的 view、api、store 文件

## 3. 色彩体系

以 `frontend/src/styles.css` 中变量为准：

| 变量 | 用途 |
|---|---|
| `--brand-red: #E60012` | Logo、主按钮、价格、激活态、重点数字、平台券重点 |
| `--brand-red-dark: #C4000F` | 主按钮 hover、强调态 |
| `--brand-red-light: #FFE8EA` | 浅红标签、轻提示背景 |
| `--bg-page: #FFFFFF` | 页面主背景 |
| `--bg-section: #F7F7F7` | 大分区背景 |
| `--bg-soft: #FAFAFA` | 次级浅背景 |
| `--bg-card: #FFFFFF` | 商品、订单、表单、优惠券容器 |
| `--text-main: #111111` | 主文字 |
| `--text-secondary: #555555` | 正文说明 |
| `--text-muted: #999999` | 辅助信息 |
| `--border-light: #EEEEEE` | 常规边框 |
| `--border-soft: #F3F3F3` | 轻边框 |

使用规则：品牌红只做行动和识别，不大面积铺底；默认按钮和 Element Plus 主色保持品牌红；商家券可以使用少量克制金色做区分，但不能把页面变成金色营销风。

## 4. 字体与层级

字体栈：`PingFang SC`、`Microsoft YaHei`、`Helvetica Neue`、`Arial`、`sans-serif`。

| 场景 | 推荐层级 |
|---|---|
| 首页 Banner 标题 | 44px-56px，粗体，行高紧凑 |
| 普通页面标题 | 30px-44px |
| 分区标题 | 26px-32px，`font-weight: 800` |
| 卡片标题 | 15px-18px，两行截断优先 |
| 正文 | 15px-17px，使用次级文字色 |
| 辅助信息 | 12px-14px，使用弱文字色 |
| 价格/优惠金额 | 品牌红、加粗，详情页或券金额可放大到 32px 左右 |

文案要清爽直接。首页可以有品牌感，功能页要告诉用户当前能完成什么，不在界面解释技术实现。

## 5. 布局系统

优先复用现有全局类：

- `.container`：主内容最大宽度约 1200px，左右留白。
- `.page`：普通页面主体，上下留白明确。
- `.section`：白底分区。
- `.section-gray`：浅灰分区。
- `.band`：轻量内容面板，用于订单、表单、个人中心等功能块。
- `.grid`：商品和内容网格。
- `.row` / `.stack`：标准横向和纵向排布。

桌面端主内容不随意突破 1200px。商品网格默认 4 列，空间充足时可调整，但不能挤压图片和标题。功能页可以更紧凑，但必须保持留白和分组。

## 6. 响应式规则

- 1080px 以下：导航、搜索和操作区可以收缩，详情页双栏开始压缩。
- 720px 以下：容器左右 16px，商品网格改为 2 列，详情页和订单页改为单列。
- 560px 以下：隐藏次要说明，按钮允许换行，标题缩小到 28px-32px。
- 固定格式元素必须设置稳定尺寸，例如商品图 `aspect-ratio: 1 / 1`，避免加载和 hover 时跳动。
- 优惠券卡片在移动端要避免金额、有效期、按钮互相挤压，可以改为纵向布局。

## 7. Header 与导航

当前 Header 是 Allmart 的主要品牌入口：白色半透明、sticky、轻边框、轻阴影、左 Logo、中导航、右搜索/购物车/用户。

标准导航顺序：

1. 首页
2. 商品分类
3. 精选推荐
4. 热门榜单
5. 直播
6. 优惠券
7. 我的订单
8. 我的售后
9. 个人中心

导航 hover 和激活态使用品牌红。搜索框、图标按钮和用户 Chip 保持圆润轻量，不增加密集促销入口。

## 8. 页面规范

| 页面 | 规范 |
|---|---|
| 首页 | 大图 Banner、平台数据、专题、分类 Tab、商品网格、楼层内容；优先读取后端真实数据 |
| 商品列表/详情 | 图片清晰，SKU、价格、库存、评价和加购动作突出；库存显示使用可售库存 |
| 购物车 | 商家分组、勾选、数量、删除和结算路径清楚，金额以接口返回为准 |
| 领券中心 `/coupons` | 只展示可领取优惠券和领取动作，不混入我的券包；可以更有活动感，但仍保持白底和品牌红点题 |
| 我的券包 `/profile/coupons` | 只展示当前用户已领取优惠券，支持状态筛选，已锁定/已使用/已过期状态弱化展示 |
| 确认订单 | 地址、商品、优惠券、运费、应付金额分区明确，不信任前端自算金额 |
| 订单中心 | 状态、商品快照、支付金额、履约动作清晰；父订单和子订单关系不要混淆 |
| 个人中心 | 资产、优惠券、收藏、浏览历史、评价入口保持统一卡片和列表节奏 |
| 推荐/榜单/直播 | 可以更有内容感，但仍要服务商品转化和真实接口数据 |

## 9. 组件规范

- 商品卡片：图片 1:1，白底，轻边框，hover 可有轻微阴影；价格品牌红，标题最多两行。
- 优惠券卡片：金额突出，门槛和有效期清楚；平台券红色主调，商家券少量金色点题，不可用券灰化并说明原因。
- 按钮：主按钮红底白字，次按钮白底红字或浅边框；不要恢复 Element Plus 默认蓝色。
- 标签：状态标签使用轻背景和短文案，避免彩色堆叠。
- 表单：字段对齐、错误提示清楚，提交按钮靠近操作区域。
- 空状态：说明当前没有数据，并提供自然下一步，例如去逛逛、去领券、返回订单。
- 弹窗：只用于确认、编辑、上传等短流程，不承载整页复杂业务。

## 10. 禁止事项

- 复制 MINISO 的 Logo、品牌名、IP、门店、加盟和新闻栏目。
- 把用户端改成后台系统、课程作业页、玻璃拟态页或大红促销页。
- 大面积红色背景、过重阴影、卡片套卡片、促销角标堆叠。
- 为了展示效果硬编码假商品、假订单、假评论、假优惠券。
- 在多个页面分散写请求头、接口地址或临时 mock 数据。
- 改动风格后不更新本文件。
