# AllMart 三端统一设计规范

## 1. 规范目标

本规范用于统一 AllMart 用户端、商家端、平台端的视觉与交互风格。当前用户端已经形成稳定的品牌基准：白底、红黑主题、轻卡片、充足留白、品牌官网感。商家端和平台端后续应以此为同一产品体系的管理端延展，不能继续保持传统后台、复杂顶栏或大屏 BI 风格。

本规范基于以下内容整理：

- 用户端前端代码：`Shopping/user/frontend/src/App.vue`
- 用户端全局样式：`Shopping/user/frontend/src/styles.css`
- 用户端代表页面：`HomeFixed.vue`、`CouponCenter.vue`、`ProductCard.vue`
- 商家端现状：`Shopping/merchant/frontend/src/App.vue`、`Dashboard.vue`
- 平台端现状：`Shopping/admin/src/layout/MainLayout.vue`、`Dashboard.vue`、`assets/styles/brand.css`
- 参考文档：`/Users/tanruiyang/Downloads/allmart-style-guide-v2.md`
- 参考图一：商家中心预期
- 参考图二：平台中心预期

## 2. 用户端基准特征

用户端不是传统商城后台，也不是强促销低价页面，而是偏品牌官网式购物体验。商家端、平台端要继承这种气质，而不是复制用户端页面结构。

### 2.1 页面气质

- 页面主背景以白色为主，浅灰只用于分区或弱背景。
- 红色只用于品牌识别、主操作、价格、当前态、重点数据和少量状态提示。
- 黑色文字用于标题和核心数字，灰色文字用于解释、时间、说明。
- 卡片使用白底、浅边框、轻阴影，避免厚重后台面板。
- 页面节奏清爽，模块之间有明显留白，不做密集数据堆叠。
- 首页和内容页有品牌型 Hero 区，管理端首页也应使用 Hero 建立统一入口。

### 2.2 Header 特征

用户端 Header 来自 `App.vue`，核心特征如下：

- Sticky 顶部导航，白底微透明，有轻边框和轻阴影。
- 左侧 AllMart Logo，中间横向菜单，右侧搜索、购物车/通知、头像身份区。
- 导航文字不大，当前态用品牌红。
- 搜索框是浅灰胶囊。
- 图标按钮是轻量圆形或无厚背景。
- Footer 保持浅灰分区、Logo 和基础链接信息。

商家端和平台端应复用这种「横向品牌导航」关系，但菜单内容换成各自业务入口。

### 2.3 用户端组件语言

用户端全局样式中已经形成一套稳定组件语言：

- `container/page`：主内容最大宽度约 1220px，左右 24px 安全边距。
- `allmart-page-hero`：白到浅灰/浅红的柔和 Hero，右侧淡红光晕。
- `allmart-chip`：胶囊筛选，选中态红色文字和浅红背景。
- `allmart-card`：白底、浅边框、16px 左右圆角、轻阴影。
- Element Plus 默认蓝色被统一覆盖为品牌红。
- 商品卡片使用 1:1 图片、轻灰图片底、15px 左右标题、红色价格、小尺寸操作按钮。
- 优惠券卡片使用白底、浅红金额区域、虚线分隔和胶囊操作按钮。

后续商家端、平台端不要直接复制用户端组件代码，但要重建同一套视觉规则和交互状态。

## 3. 三端视觉关系

### 3.1 用户端

定位：品牌购物官网。

关键词：

- 精选商品
- 购物链路
- 轻卡片
- 内容浏览
- 品牌氛围

用户端保持现状为主，仅在必要时微调全局变量和组件细节。

### 3.2 商家端

定位：经营工作台 + 数据辅助决策。

关键词：

- 店铺经营
- 订单处理
- 库存预警
- 客服消息
- 营销管理

商家端应像用户端品牌体系下的经营中心，而不是另一个红色淘宝后台。参考图一的方向：顶部品牌导航、办公场景 Hero、五个核心指标卡、图表卡片、待处理订单、库存预警、运营预警、客服消息、热销商品。

### 3.3 平台端

定位：平台治理 + 运营监管中心。

关键词：

- 商家监管
- 订单监督
- 用户管理
- 风险预警
- 平台数据
- 服务市场

平台端应比商家端更偏治理和全局视角，但仍然保持白底轻卡片。参考图二的方向：顶部品牌导航、平台中心 Hero、五个全局指标卡、交易趋势、订单状态、品类成交、待处理事项、风险预警、运营公告、重点商家。

## 4. 全局设计 Token

三端建议共用同名变量。若三端无法共享同一 CSS 文件，也应保持数值一致。

| Token | 值 | 用途 |
|---|---:|---|
| `--brand-red` | `#E60012` | Logo、主按钮、激活态、价格、重点数字 |
| `--brand-red-dark` | `#C4000F` | Hover、强强调 |
| `--brand-red-light` | `#FFE8EA` | 浅红标签、选中背景 |
| `--bg-page` | `#FFFFFF` | 页面背景 |
| `--bg-section` | `#F7F7F7` | 分区和 Footer |
| `--bg-soft` | `#FAFAFA` | 输入框、弱背景 |
| `--bg-card` | `#FFFFFF` | 卡片背景 |
| `--text-main` | `#111111` | 标题、核心文字 |
| `--text-secondary` | `#555555` | 正文、说明 |
| `--text-muted` | `#999999` | 时间、弱说明 |
| `--border-light` | `#EEEEEE` | 卡片和表格边框 |
| `--border-soft` | `#F3F3F3` | 更弱分割线 |
| `--shadow-soft` | `0 8px 24px rgba(0,0,0,0.05)` | 普通卡片阴影 |
| `--shadow-card` | `0 12px 32px rgba(0,0,0,0.08)` | 悬浮或重点卡片 |
| `--radius-lg` | `16px` | 卡片、大容器 |
| `--radius-md` | `10px` | 小卡片、图片 |
| `--radius-sm` | `6px` | 小控件 |
| `--radius-pill` | `999px` | 按钮、筛选、标签 |

色彩使用原则：

- 品牌红不大面积铺底。
- 主趋势图使用红色，辅助分类图可用蓝、绿、橙、灰，但单张图不超过 6 色。
- 商家端和平台端不要使用 Element Plus 默认蓝作为主色。
- 不使用深色大屏背景，不使用高饱和渐变块。
- 不使用大面积橙色、紫色、蓝灰色作为页面主调。

## 5. 字体和信息层级

字体栈：

```css
"PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif
```

推荐层级：

| 场景 | 字号 | 字重 | 用法 |
|---|---:|---:|---|
| 管理端 Hero 标题 | 44px - 52px | 800 | 商家中心、平台中心 |
| 页面标题 | 28px - 36px | 800 | 商品管理、订单管理等 |
| 卡片模块标题 | 16px - 20px | 700/800 | 图表卡、列表卡 |
| 指标名称 | 13px - 15px | 600/700 | 指标卡说明 |
| 指标大数字 | 26px - 34px | 800 | GMV、订单数等 |
| 表格正文 | 13px - 14px | 400/500 | 数据行 |
| 辅助信息 | 12px - 13px | 400 | 时间、说明、同比 |

注意：

- 不要所有文字都加粗。
- 表格页面的标题不要用 Hero 级字号。
- 按钮、标签、表格操作文字要保证移动端不挤压。

## 6. 三端统一布局

### 6.1 主体宽度

- 用户端：保持约 1220px。
- 商家端首页/平台端首页：建议 1280px，参考图一图二的仪表盘密度。
- 普通管理页：1200px - 1280px。
- 所有页面左右至少 24px 安全边距，移动端至少 16px。

### 6.2 标准页面结构

三端统一为：

1. Header
2. Hero 或 Page Header
3. 主内容区
4. Footer

商家端和平台端当前不建议继续使用多层顶栏、侧栏、右侧悬浮工具栏的复杂布局。应改为参考图中的单层横向导航工作台。

### 6.3 分区节奏

- Hero 后接指标卡。
- 指标卡后接 3 列图表区。
- 中部用 2:1 或 2:1:1 的卡片组合承载待办、预警、分析。
- 底部可放消息、热销商品、重点商家等列表/卡片。
- 卡片间距桌面端 20px - 24px，管理端不要小于 16px。

## 7. Header 规范

### 7.1 三端共同规则

- 白底或 96% 以上透明白底。
- 高度 72px - 76px。
- Sticky 顶部。
- 底部 1px 浅灰边框，可加很轻阴影。
- 左侧使用 AllMart 完整 Logo，不手绘新 Logo。
- 中间使用横向导航，不使用传统左侧深色侧栏。
- 当前导航：红色文字 + 红色短下划线。
- 右侧：搜索、通知、头像、身份名称、下拉箭头。
- 搜索框：浅灰胶囊，高 36px - 42px。

### 7.2 商家端导航

推荐导航：

1. 首页
2. 商品管理
3. 订单管理
4. 营销中心
5. 数据中心
6. 店铺管理
7. 财务中心
8. 服务市场

可按路由映射：

- 商品管理：商品、库存、评价申诉
- 订单管理：订单、售后、物流
- 营销中心：优惠券、活动、直播
- 店铺管理：店铺设置、入驻审核
- 服务市场：平台客服、客服管理

### 7.3 平台端导航

推荐导航：

1. 首页
2. 商家管理
3. 订单监督
4. 用户管理
5. 营销管理
6. 数据中心
7. 财务中心
8. 服务市场

可按路由映射：

- 商家管理：商户、商品、分类
- 订单监督：订单、售后、纠纷、异常
- 营销管理：优惠券、活动、轮播
- 数据中心：报表、日志、运营指标
- 服务市场：客服、通知、AI 助手

## 8. 商家端首页规范

商家端首页目标效果参考图一。

### 8.1 Hero

内容结构：

- 左侧：
  - 大标题：商家中心
  - 欢迎语：欢迎回来，AllMart旗舰店
  - 说明：实时掌握店铺经营数据，助力生意增长
- 右侧：
  - 淡雅办公/商品场景图，如托特包、笔记本电脑、咖啡、绿植。

视觉要求：

- 高度 250px - 300px。
- 背景以白色为主，右侧图片自然融入。
- 图片不要深色、不要强对比、不要抢文字。
- Hero 下方立即接 5 个指标卡。

### 8.2 五个核心指标

推荐指标：

1. 今日营业额
2. 支付订单数
3. 访客数
4. 转化率
5. 退款率

卡片结构：

- 顶部红色线性图标。
- 指标名称 13px - 14px。
- 大数字 26px - 30px。
- 底部较昨日涨跌，红色表示增长，绿色表示下降或风险降低。

视觉要求：

- 5 张一行，等宽。
- 白底、浅边框、18px - 20px 圆角。
- 阴影很轻。
- 卡片不要使用大面积浅红背景。

### 8.3 商家首页模块

第一行图表区：

- 营业额趋势：红色折线/面积图。
- 订单时段分布：红色柱状图，可用浅红渐变。
- 商品分类销售占比：环图，红色为主分类，蓝绿橙灰为辅助。

第二行运营区：

- 待处理订单：表格卡片，支持全部/待付款/待发货/退款售后 Tab。
- 库存预警：小图 + 商品名 + 库存不足数量。

第三行分析区：

- 运营预警：规则/建议列表。
- 新老客分析：环图 + 新客/老客比例。
- 用户画像分析：性别条形、年龄胶囊。

第四行内容区：

- 客服消息：头像、用户、摘要、时间、未读点。
- 热销商品：横向商品卡片，图片 1:1，价格红色，销量灰色。

### 8.4 商家端需要移除或收敛的现状

当前 `Shopping/merchant/frontend/src/App.vue` 中不建议保留：

- 顶部 32px 小导航 + 96px 主 Header + 二级横向菜单三层结构。
- 右侧固定快捷工具条。
- 大红描边搜索框。
- 表情符号作为工具图标。
- “商家工作台 / 商品入口 / 活动入口”这类临时入口文案。

改造方向：

- 合并为单层品牌 Header。
- 菜单按业务分组保留 8 个以内一级入口。
- 快捷动作放入首页卡片或 Header 通知，不做右侧悬浮栏。
- 所有图标使用 Element Plus 图标或同风格线性图标。

## 9. 平台端首页规范

平台端首页目标效果参考图二。

### 9.1 Hero

内容结构：

- 左侧：
  - 大标题：平台中心
  - 欢迎语：欢迎回来，AllMart平台管理员
  - 说明：统一掌握商家、订单与平台运营情况，提升管理效率
- 右侧：
  - 办公/平台运营场景图，可与商家端同一系列但不完全相同。

视觉要求：

- 平台端可比商家端更偏办公监管，但仍保持温和、明亮。

### 9.2 五个核心指标

推荐指标：

1. 平台 GMV
2. 支付订单数
3. 活跃商家数
4. 活跃用户数
5. 投诉率

结构和视觉同商家端指标卡。

### 9.3 平台首页模块

第一行图表区：

- 平台交易趋势：红色折线/面积图。
- 订单状态分布：横向条形图，用蓝/红/绿/橙区分状态。
- 品类成交占比：环图。

第二行治理区：

- 待处理事项：商家入驻审核、售后/投诉、违规处理、工单消息。
- 风险预警：异常退款率、库存异常、商家评分下降、违规商品预警。

第三行运营区：

- 运营公告：公告标题、摘要、日期。
- 商家增长分析：环图 + 新入驻/活跃/沉睡商家。
- 用户画像分析：性别和年龄分布。

第四行内容区：

- 平台消息：商家/用户/系统消息。
- 重点商家：横向商家卡片，展示店铺图、状态标签、月销售额、环比。

### 9.4 平台端需要收敛的现状

当前 `Shopping/admin/src/layout/MainLayout.vue` 已经接近横向品牌导航，但仍需调整：

- Logo 应改为与用户端一致的图片资产，而不是内嵌手绘 SVG。
- 导航命名要从“商品/订单/商户/用户”收敛到参考图的业务中心命名。
- Footer 文案不要偏用户端营销，应改为平台帮助、平台规则、隐私政策、联系我们。
- Dashboard 中大量角色工作台、彩色 emoji、渐变圆形装饰要收敛为统一卡片和业务模块。
- 风控、客服、财务角色差异可保留在内容数据上，不改变整体页面骨架。

## 10. 卡片规范

### 10.1 普通卡片

适用：图表、列表、待办、公告、消息。

要求：

- 白底。
- 1px 浅灰边框。
- 16px - 20px 圆角。
- 轻阴影。
- 内边距 20px - 24px。
- 标题左上，更多/查看全部右上。

### 10.2 指标卡

适用：首页 5 个核心指标。

要求：

- 高度 150px - 170px。
- 图标 26px - 32px，红色线性。
- 数字最多两行以内，金额需格式化。
- 同比区域放底部，12px - 13px。

### 10.3 图表卡

要求：

- 图表高度 260px - 320px。
- 坐标轴、网格线使用浅灰。
- 主趋势色使用品牌红。
- Tooltip 白底、轻阴影、圆角。
- 卡片标题不超过 18px。
- 不使用黑底、霓虹色、厚重渐变。

### 10.4 列表卡

适用：待处理订单、待处理事项、库存预警、风险预警、消息、公告。

要求：

- 每行高度稳定，通常 54px - 72px。
- 商品/商家图片 44px - 64px。
- 头像 32px - 40px。
- 行之间用浅灰分割线。
- 操作按钮使用小胶囊。
- 状态标签用浅红、浅灰、浅绿，不要大色块。

## 11. 表格规范

商家端和平台端普通功能页会大量使用表格，需保持品牌统一。

要求：

- 表格外层使用白色卡片。
- 表头浅灰背景或白底加分割线。
- 表头文字 12px - 13px，灰黑色，600 字重。
- 行高 48px - 56px。
- Hover 背景使用极浅红或极浅灰。
- 操作按钮为小胶囊：处理、查看、发货、审核。
- 危险操作使用红色描边，不直接使用大红实心。
- 不使用粗边框，不使用蓝色选中态。

## 12. 按钮和筛选

### 12.1 主按钮

用途：

- 保存
- 发布
- 发货
- 处理
- 审核通过
- 发送

视觉：

- 红底白字。
- 胶囊圆角。
- 高度 34px - 42px。
- Hover 使用深红。

### 12.2 次按钮

用途：

- 查看详情
- 筛选
- 返回
- 取消
- 加入购物车类弱操作

视觉：

- 白底。
- 浅灰或浅红边框。
- 文字灰色或红色。
- Hover 变浅红背景。

### 12.3 Tab / Chip

要求：

- 胶囊或红色下划线二选一。
- 首页模块和筛选用胶囊。
- 表格状态 Tab 可用红色文字 + 下划线。
- Badge 数字使用红底白字，小尺寸。

## 13. 图标规范

- 使用线性图标，优先 Element Plus 图标库；后续如引入 lucide，也需三端统一。
- 指标卡图标统一红色。
- 辅助图标使用黑色或灰色。
- 不使用 emoji 做正式业务入口。
- 图标尺寸：
  - Header：18px - 20px
  - 指标卡：28px - 32px
  - 列表图标：18px - 24px

## 14. 图片与资产规范

### 14.1 Logo

三端统一使用用户端已有品牌资产：

- `Shopping/user/frontend/public/brand-assets/allmart-logo-full.png`
- 或在各端复制同名资产并保持一致。

不建议各端自行手写 SVG Logo 或用纯文字 Logo。

### 14.2 Hero 图片

商家端和平台端参考图一图二，应使用同一摄影风格：

- 明亮白底。
- 办公桌、笔记本、托特包、咖啡、绿植、产品样品。
- 低饱和、干净、有生活感。
- 右侧展示，不干扰左侧文字。
- 不使用深色科技图、抽象渐变、低质库存图。

### 14.3 商品/商家图片

- 商品图保持 1:1。
- 列表缩略图 48px - 64px。
- 重点商家图可使用 4:3 或 1:1，但同一模块必须统一比例。
- 图片容器使用浅灰底和 8px - 12px 圆角。

## 15. Element Plus 覆写规则

三端都使用 Vue + Element Plus，应统一覆盖默认主题：

- `--el-color-primary` 映射到 `--brand-red`。
- Primary 按钮使用品牌红。
- Select/Input Focus 使用品牌红边框。
- Tabs active bar 使用品牌红。
- Pagination active 使用品牌红。
- Checkbox/Radio/Switch checked 使用品牌红。
- Badge 使用品牌红。
- Table hover 使用极浅红或极浅灰。
- 不允许出现默认蓝色主按钮、蓝色 active tab、蓝色 focus 边框。

## 16. 响应式规范

桌面端：

- Header 横向显示。
- 首页指标 5 列。
- 图表区 3 列。
- 商品/重点商家卡片 4 列。

中等屏幕：

- 指标可变为 3 + 2。
- 图表区变为 2 列。
- 表格允许横向滚动。

移动端：

- Header 菜单横向滚动或折叠菜单。
- 指标卡 2 列。
- 图表和列表单列。
- Hero 图片弱化或隐藏，仅保留淡背景。
- 所有按钮文字不能溢出。

## 17. 禁止事项

- 禁止商家端和平台端继续使用传统深色侧边栏。
- 禁止使用大屏 BI 黑底、霓虹色、强渐变。
- 禁止大面积红色背景。
- 禁止 Element Plus 默认蓝色主色。
- 禁止 emoji 作为正式业务图标。
- 禁止卡片套卡片过多。
- 禁止页面贴边或无限拉宽。
- 禁止大量临时入口文案和重复导航。
- 禁止不同端使用不同 Logo 造型。

## 18. 后续改造优先级

### 第一阶段：统一骨架

1. 商家端改成单层品牌 Header。
2. 平台端 Header Logo 和菜单命名统一。
3. 三端统一 Token 和 Element Plus 覆写。
4. 商家端、平台端首页增加 Hero + 5 指标卡骨架。

### 第二阶段：改首页内容

1. 商家端 Dashboard 改为图一结构。
2. 平台端 Dashboard 改为图二结构。
3. 图表统一红色主趋势和浅灰坐标。
4. 预警、消息、订单、库存、商家列表统一卡片。

### 第三阶段：改普通功能页

1. 商品管理、订单管理、售后、商户管理、用户管理表格统一。
2. 表单页、详情页、弹窗统一按钮和输入框。
3. 消息/客服页面统一聊天壳和卡片状态。
4. 移动端补齐响应式。

## 19. 验收清单

每个改造页面上线前检查：

- 是否使用 AllMart 统一 Logo。
- 是否白底为主。
- 是否没有默认蓝色主色。
- 当前导航是否红色激活。
- 主按钮是否红底胶囊。
- 卡片是否白底、浅边框、轻阴影。
- 图表是否克制、主趋势为红。
- 表格行高和操作按钮是否统一。
- 页面是否没有深色大屏感。
- 手机宽度下文字和按钮是否不溢出。
